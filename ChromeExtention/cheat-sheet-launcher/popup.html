操作・コマンド 詳細・リカバリ
git init新規リポジトリの初期化。
.git フォルダを作成して管理を開始する。
git clonegit clone [url]
既存のリモートリポジトリをローカルに複製する。
git status現在の変更状況を確認。
「困ったらまず実行」で状況を把握する。
git addgit add .
変更をステージングエリア（コミット準備）に追加する。
git restoregit restore file.txt
作業ディレクトリの変更を取り消す（addする前）。
git restore --stagedgit restore --staged file.txt
addを取り消してステージングから降ろす。
git reset HEADgit reset HEAD file.txt
addを取り消す従来の方法。最近はrestore推奨。
git commitgit commit -m "msg"
ステージングした変更を履歴に記録する。
git commit --amend直前のコミット内容やメッセージを修正する。
Push済みには使用禁止。
git reset --soft HEAD^直前のコミットだけ取り消し、変更はステージングに残す。
git reset --mixed HEAD^直前のコミットとステージングを取り消し、変更はファイルとして残す（デフォルト）。
git reset --hard HEAD^直前のコミットと変更を完全に削除する。
危険なため実行前に確認推奨。
git reset --hard <commit>特定のコミットまで履歴を強制的に戻す。その後の履歴は消える。
git revertgit revert [commit]
履歴を残したまま、指定コミットを打ち消すコミットを作成する。安全。
git reflogHEADの移動履歴を表示。
reset --hard で消えたコミットもここから探して復元できる最強のリカバリ手段。
git reset --hard HEAD@{n}reflogで見つけた過去の状態へ復元する。
git stash作業を一時退避する。
git stash pop退避した作業を復元する。
git stash list退避した作業一覧を表示する。
git branchブランチ一覧表示・作成。
git switchブランチの切り替え。
git switch -c new-branch で作成と切り替えを同時に行う。
git checkoutブランチ切り替え（従来版）。
git merge別ブランチを統合する。
コンフリクト時は手動修正してコミットする。
git logコミット履歴を確認する。
git log --oneline --graphコミット履歴をグラフで視覚的に確認する。
git diff変更差分を表示する。
操作・作業 詳細補足・リカバリ
リポジトリの作成New Repositoryから作成。Public/Privateを選択。間違えてもSettingsから削除可能。
ファイルの作成・編集Web上の「Add file」や鉛筆アイコンから直接編集可能。PreviewタブでMarkdown確認。
ファイル編集中の変更破棄【リカバリ】 編集画面で「Cancel changes」を押せばコミット前に変更を破棄できる。
ブランチの作成ファイル一覧左上のブランチ選択ボタンに新しい名前を入力して作成。
プルリクエストの作成変更をmain等に統合する依頼。「Compare & pull request」から作成。
Issueの作成バグ報告やタスク管理に使用。LabelやAssigneesを設定すると管理しやすい。
コミット履歴の確認「commits」リンクから履歴一覧を確認。各コミットをクリックで詳細差分を表示。
プルリクエストのマージレビュー完了後「Merge pull request」ボタンで統合。競合時は解消が必要。
マージ後のブランチ削除マージ後の「Delete branch」で削除して整理する。
誤って削除したブランチの復元【リカバリ】 PR画面やBranches一覧の「Restore」ボタンで復元可能。
閉じたIssue/PRの再開【リカバリ】 誤ってCloseした場合、下部の「Reopen」ボタンで即座に再開できる。
マージ済みPRの取り消し (Revert)【リカバリ】 マージ済みPR画面の「Revert」ボタンを押すと、変更を打ち消すPRが自動作成される。
リポジトリのフォーク (Fork)他人のリポジトリを自分のアカウントにコピーして編集可能にする。
リポジトリのクローン (Clone)緑色のCodeボタンからURLを取得し、ローカルPCにダウンロードする。
リポジトリの削除Settings最下部「Danger Zone」から削除。復元困難なため要注意。
操作・ショートカット 詳細・リカバリ
タイムライン機能 (左サイドバー) 【重要】 ファイルの保存履歴が自動記録されており、誤って上書きしても過去の状態に戻せる。
Ctrl+S / Cmd+S 保存。基本中の基本。こまめな保存がクラッシュ時の被害を防ぐ。
Ctrl+Shift+S / Cmd+Shift+S 名前を付けて保存。バックアップとして別名保存する際などに使用。
Ctrl+Shift+T / Cmd+Shift+T 【リカバリ】 誤って閉じたタブを再表示する。連打すれば履歴順に復元可能。
Ctrl+K Ctrl+W / Cmd+K Cmd+W すべてのタブを閉じる。誤操作注意。閉じた場合は上記のショートカットで復元。
Ctrl+Shift+K / Cmd+Shift+K 行の削除。カーソル行を一発で削除する。
Alt+↑↓ / Opt+↑↓ 行の移動。コピペせずに行を入れ替えられる。
Alt+Shift+↑↓ / Opt+Shift+↑↓ 行の複製。現在の行をそのまま下にコピーする。
Ctrl+F / Cmd+F ファイル内検索。誤って消した箇所を探すのにも使える。
Ctrl+H / Cmd+Opt+F 置換。一括修正用。プレビューで確認してから実行推奨。
Ctrl+Shift+F / Cmd+Shift+F 全体検索。プロジェクト内の全ファイルから文字列を探す。
Ctrl+P / Cmd+P クイックオープン。ファイル名を入力して素早く開く。
Ctrl+G / Cmd+Ctrl+G 指定行へジャンプ。エラーログの行番号へ飛ぶのに便利。
Ctrl+/ / Cmd+/ コメントアウト。削除せずに無効化したい時に使う。
Ctrl+B / Cmd+B サイドバー表示切替。画面を広く使いたい時に。
Ctrl+Shift+P / Cmd+Shift+P コマンドパレット。全機能を検索・実行できる万能コマンド。
ゴミ箱からの復元 VS Codeで削除したファイルはOSのゴミ箱に入るのでそこから復元可能。
Local History 拡張 【推奨】 拡張機能を入れておくと、保存ごとのバックアップ履歴が残る。
操作・作業 詳細・リカバリ
シンプルな表（テーブル）を作るディメンションと指標をドラッグ＆ドロップ。表示がおかしい時は一度クリアして再設定。
スコアカードで合計・平均を表示重要KPIを一目で把握。「集計方法」（合計・平均・カウントなど）の設定ミスに注意。
折れ線グラフで時系列を可視化ディメンションに日付を設定。粒度（日・週・月）は詳細設定で変更可能。
フィルタコントロール追加期間やプルダウンで条件切り替え。効かない場合は「適用範囲」やデータソースを確認。
レポートレベル vs グラフレベルフィルタの適用範囲の違い。誤って全体に効かせた場合は「レポートレベルから削除」で戻す。
ページの追加・複製似たレポートを作る際はページごと複製し、データソースだけ差し替えると効率的。
グラフのコピペとデータ差し替え既存グラフをコピーして別データのプロパティに変更。値がおかしい時は元に戻して確認。
計算フィールドでCTR/CVRを作るClicks / Impressions など独自の指標を作成。分母0のエラーはIF関数で回避。
CASE式でカテゴリ分け条件に応じたランク分けやグルーピングを作成。NULL対策にELSEを用意する。
並べ替え（ソート）設定重要な順にデータを並べる。意図しない順序の時はソート対象フィールドを確認。
データコントロール追加GA4などのプロパティを閲覧者が切り替えられるようにする。フィールド共通化に注意。
デフォルト集計方法の確認データソース側でSUM/AVGなどが正しく設定されているか確認する。
レポートの共有設定と権限閲覧者と編集者を分け、誤編集を防ぐ。誤った場合はバージョン履歴で戻す。
編集モード／表示モードの切替誤操作防止のため、閲覧時は必ず表示モードにする。
統合（混合データ）の基本設定複数データソースをキー（日付など）で結合。重複や欠損時は結合タイプを見直す。
不要なフィールド・統合の整理使わない計算フィールド等は削除して軽量化。削除ミスは「元に戻す」で対応。
操作・作業 詳細・リカバリ
1.ブロックの基本と「/」コマンド 「/」と打つとメニューが開き、種類を選んで追加。間違えたら「Turn into」で変換可能。
2.ページ履歴（バージョン履歴）を確認 【重要】 右上の時計アイコンから履歴を確認・復元。誤消去時の対策。
3.検索でページや失った情報を探す Cmd/Ctrl + P。ゴミ箱を見る前にまず検索でタイトル・本文を探す。
4.テンプレートギャラリーを利用 公式テンプレートを複製して使うと、元を壊す心配なく編集できる。
5.見出し・区切り線でページを整理 「/heading」「/divider」で構造化。「Turn into」で後から見出しレベル変更も可能。
6.To-doリストでタスク管理 完了タスクは消さずにチェックで残すと履歴になる。誤削除防止にもなる。
7.シンプルなテーブルデータベース インラインテーブルで簡易DB作成。列を増やしすぎて混乱したら履歴から戻す。
8.プロパティ（列）を追加・編集 列の型変更はデータ消失のリスクがあるため、変更前にページ複製でテスト推奨。
9.ソートとフィルター 「データが消えた」と思ったらフィルターを確認。解除すると表示されることが多い。
10.ビューの切り替え 同じDBをボードやカレンダーで表示。ビューを削除してもデータは消えない。
11.ページリンクと@メンション 「@ページ名」でリンクを作成。ハブページを作って整理すると迷子になりにくい。
12.サイドバー整理とお気に入り ドラッグで階層化。場所が不明になったら検索し、トップレベルに戻して整理。
13.トグル（折りたたみ） 情報を隠してスッキリさせる。消えたと思ったらトグルが開いているか確認。
14.埋め込み（Embed） 外部サイトや動画を埋め込み。表示されない場合はリンクとして残す。
15.共有設定と公開リンク Web公開のオンオフ確認。誤公開時は即座にオフにすればアクセス不可になる。
16.ページの複製（Duplicate） DB改造やレイアウト変更の実験は、複製したページで行うと安全。
17.インラインDBとフルページDB 複雑になったインラインDBはフルページに移し替え、リンクドビューで表示すると整理しやすい。
18.同期ブロック（Synced block） 複数箇所の記述を同期。解除時は「Unsync」しないと全箇所消えるので注意。
19.モバイルアプリと同期の注意 電波不安定時の編集はラグがある。重要な編集は安定した回線で行う。
20.エクスポートでバックアップ Markdown/HTML形式で定期的にエクスポートし、ローカルに保存しておく。
21.「シンプルを保つ」設計のコツ 最初はDB数を絞る。複雑すぎるDBは作り直す前に小さな整理を繰り返す。

操作・目的 詳細・リカバリ
タイムゾーン設定Cron等がズレないよう、Workflow設定でJST等に合わせておく。
手動実行（Execute workflow）その場で実行テスト。エラー時は該当ノードを確認し修正・再実行。
単一ノード実行特定ノードだけテスト実行。前処理が重い場合に効率的。
トリガーの種類の理解Webhook(外部呼出)、Cron(時間)、アプリトリガー(イベント)を使い分ける。
Webhook Trigger設定テスト時はListenモードで受信確認。本番URLとの切り替えに注意。
Cron Trigger設定指定時間に動かない場合はタイムゾーンと有効化スイッチを確認。
アプリトリガー検知検知されない時は認証、権限、対象フォルダ等を再確認し作り直す。
HTTP Requestノード任意APIを実行。4xx/5xxエラー時は設定値とAPI仕様を照合。
クレデンシャル作成認証情報を作成・選択。エラー時は再認証やキー再発行を行う。
クレデンシャルエラー対処Invalid表示時は、再接続テストを行い、NW設定等も確認する。
Pin data (データ固定)サンプルデータを固定し、トリガー再実行なしで下流ノードを調整。
Items (配列) の理解データが増減する場合は実行ビューで確認し、不要なループを制御。
Setノードで整理必要なフィールドのみ残す、変数を定義する等でデータ構造を整える。
Expressions (式)前ノードの値を動的に参照。エラー時は固定値で確認後、パスを修正。
IF / Filter条件分岐条件が効かない時は、型（数値/文字列）や入力値を再確認する。
Switchノード分岐複数ルートへの分岐。新ルート追加時は既存への影響を確認する。
Merge / Combineデータ結合。Primary/Secondaryや結合キーの設定ミスに注意。
エラー時の動作設定「Continue on Fail」で停止を防ぐ。後で原因究明が必要。
実行ログ (Executions)過去の成功・失敗履歴を確認し、いつから動かなくなったか特定。
ワークフロー有効化本番稼働スイッチ。不具合時は即無効化し、コピー版で修正テスト。
テンプレート活用公式例を読み込み、構造を真似る。理解のためコメントを残すと良い。
エクスポートでバックアップ【リカバリ】 修正前は必ずJSONでエクスポートして保存する。
環境変数・秘密情報APIキーは直接書かずクレデンシャルや環境変数で管理しリスク低減。
